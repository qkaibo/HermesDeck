✅ 核心优势：PilotDeck 原生能力全保留
SessionRouter 的 beginTurn/endTurn 提供同 session 并发保护（session_busy）。
TurnRunner 自动维护 JSONL transcript，进程重启后 resumeAgentSession() 从磁盘重建。
这些能力 v1 的 gRPC monkey-patch 方案全部绕过 → 手写替代品 → 漏洞百出。
⚠️ 已知限制：WebSocket 并发竞态窗口
GatewayWsConnection.handleMessage 用 void 丢弃 async Promise。
极端情况下两条消息可能在 turn_completed 和 endTurn() 之间到达 → session_busy 错误。
真实用户场景不会发生（用户有思考时间），测试时可加 100ms delay。

⚠️ 旧 JSONL 兼容性问题
v1 (gRPC) 写入的 transcript 缺少 permissionDenials 字段。
v2 进程重启时 TranscriptReplay.cloneTurnResult() 调用 result.permissionDenials.map(...) → 崩溃。
修复：删除问题 JSONL，或让 v2 TurnRunner 重新生成完整 turn_result（含 permissionDenials: []）。

1
Browser WebSocket → GatewayWsConnection.handleMessage() — 接收消息体
2
SessionRouter.beginTurn() — 检查 inFlightTurns，设置并发锁
3

4

5
http_bridge.extract_user_message() → wrapper.process() → AIAgent.run_conversation() (独立线程)
6
AIAgent → DeepSeek API (HTTPS) → text 回调 → queue → SSE chunk → TurnRunner
7
TurnRunner 写 JSONL transcript → 更新 state.messages → endTurn() 释放锁

📜 架构演进对比

❌ v1 — gRPC + Monkey-Patch (已废弃)

Gateway 猴补丁 → gRPC → Python Sidecar

• 劫持 InProcessGateway.submitTurn
• 绕过 beginTurn → 自定义锁 → 并发 bug
• 绕过 transcript → 手写 JSONL → 竞态条件
• 绕过 state.messages → 手写 history 字典
• 手写 _session_locks / _cleanup_loop

✅ v2 (当前) — HTTP/SSE + 原生 Provider

PilotDeck 原生 → HTTP/SSE → Python Bridge

• beginTurn/endTurn — 原生并发保护
• SessionRouter — 30min 空闲清理
• TurnRunner — JSONL 自动写盘
• state.messages — 对话历史自动维护
• Sidecar 只是 LLM provider — 零副作用
关键教训：不要 monkey-patch 框架核心流程
三次尝试劫持 submitTurn，三次失败。最终方案：修改 pilotdeck.yaml 的 provider URL，让 PilotDeck 以为是调用 DeepSeek，实际走到 Python Bridge。框架能力全部保留，Sidecar 只做一件事——当 LLM API。

⚙️ 配置铁律

pilotdeck.yaml (关键字段)

# ~/.hermesdeck/pilotdeck.yaml agent: model: hermesdeck/deepseek-v4-flash ← provider/model 格式 model.providers.hermesdeck: protocol: openai url: http://localhost:29553/v1 ← 不要加 /chat/completions apiKey: hermesdeck-bridge-token router.tokenSaver.enabled: false ← 必须关！

常见配置错误

1 url 末尾加 /chat/completions → 路径重复 → 404
2 tokenSaver.enabled: true → 简单问题路由到弱模型
3 agent.model 格式不是 provider/model → 解析失败
4 provider URL 写成 http://localhost:29553 (无 /v1) → PilotDeck 拼接后变成 /chat/completions，不行
5 skip_memory=True → Agent 失去用户 profile 和项目规则
6 ephemeral_system_prompt 写法不当 → Agent 逐字背诵 prompt 中的示例回答

● 架构分层

  • L1 — 客户端: Browser + React UI :28789
  • L2 — Gateway: PilotDeck 原生会话引擎 :28788
  • L3 — Bridge: Python HTTP/SSE + AIAgent :29553
  • L4 — 外部: DeepSeek API
  • L5 — 持久化: 5 种数据存储

● 关键设计

  • 🔸 透明 provider — 替换 URL 即可，框架零修改
  • 🔸 per-session 隔离 — 独立 AIAgent，线程安全
  • 🔸 自恢复 — JSONL replay 重建 session
  • 🔸 并发保护 — beginTurn/endTurn + per-session lock
  • 🔸 空闲回收 — Gateway 30min + Python 10min
